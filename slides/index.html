Let's get some input to work with!

Our strategy:

  • Store each stroke as an array of points
  • Connect the dots to draw the stroke
  • When the mouse is pressed, add a new array for a new stroke
  • When the mouse is dragged, add the new mouse position to the last stroke

Tip: smoothing

Instead of directly adding the new point, add a point 50% of the way to the new point.

WebGL???

  • What?
    • A totally separate way of drawing from 2D mode
    • Harder to use (does less for you) but much more flexible
    • Great for 3D graphics, but it can do 2D too
  • Why?
    • It lets you run a lot of calculations in parallel on the GPU
    • Gives you the freedom to write your own rendering algorithms and have them be fast
  • How?

How?

WebGL gives you two spots where you can add custom rendering code via shaders:

Shape data starts in Javascript as the vertices of triangles

The vertex shader runs on each vertex in parallel calculating where on the screen it should draw

The fragment shader runs in parallel for each pixel in each triangle calculating what color it should be
